01

What You Need Before Starting

Before we dive in, let's make sure you have everything ready. You only need three simple things to get started.

  1. A Custom Domain
    Your website address like yourname.com. You can get one from GoDaddy, Namecheap, or any domain registrar. Usually costs around $10-15 per year.
  2. A Free GitHub Account
    Head over to github.com, sign up, and confirm your email. Takes about 2 minutes and costs nothing.
  3. Your Website Files
    Either I've sent you a ZIP file with your cloned site, or you have access to the GitHub repository with your website files.
Ready to Go!
Once you have all three items above, you're ready to move forward. Let's make this happen!
02

Create Your GitHub Repository

Think of a repository (repo for short) as a special folder that holds all your website files. Here's exactly how to create one.

  1. Log in to your GitHub account at github.com
  2. Look for the green "New" button (usually in the top left) and click it
  3. Choose a simple Repository name - something like yourname-site or my-crypto-site
  4. Make sure it's set to Public (this is required for free GitHub Pages hosting)
  5. Click the green "Create repository" button at the bottom
Congratulations!
You now have an empty repository where your website will live. That was easy, wasn't it? Let's keep this momentum going!
03

Upload Your Website Files

Now we're going to get your website files into GitHub. If I sent you a ZIP file, first unzip it on your computer.

  1. Open your new repository on GitHub (you should already be there)
  2. Click "Add file" then select "Upload files" from the dropdown
  3. Drag and drop ALL your website files and folders into the upload area (or click to browse and select them)
  4. Scroll down and click the green "Commit changes" button
CRITICAL: File Structure Matters
Your main file MUST be named index.html and it must be in the root (main folder) of your repository. If you have folders like /images, /css, or /js, upload those exactly as they are.
Files Uploaded Successfully
Your website code is now safely stored on GitHub and ready to be published. You're doing great!
04

Turn On GitHub Pages

This is where the magic happens! We're going to tell GitHub to publish your site and make it accessible to the world.

  1. In your repository, click on "Settings" (it's in the top menu of your repo)
  2. On the left sidebar, scroll down and click "Pages" (under "Code and automation")
  3. Under "Source" or "Build and deployment":
    • Select "Deploy from a branch"
    • Choose branch: main
    • Choose folder: / (root)
  4. Click the "Save" button
Wait 1-2 Minutes...
GitHub is building your site right now. Give it a moment, then refresh the Settings ‚Üí Pages screen. You'll see a green banner with your site URL.
Your Temporary Site is Live!
Your site should now be available at something like:
https://yourusername.github.io/yourname-site

Click that link and test it! If you see your website, you're crushing it! Now let's connect your custom domain.
05

Connect Your Custom Domain in GitHub

First, let's decide how you want your domain to work. You have two options, and I'll help you understand which one is right for you.

üî∑ Option A: Subdomain

  • Easier to set up
  • Recommended for beginners
  • Keeps your main domain free for other uses

üî∑ Option B: Root Domain

  • Cleaner, more professional look
  • Requires A records setup
  • Perfect for primary websites

Now, add your domain to GitHub:

  1. Stay in Settings ‚Üí Pages in your repository
  2. Find the "Custom domain" box
  3. Type your domain:
    • For subdomain: members.yourname.com
    • For root domain: yourname.com
  4. Click "Save"
Keep This Page Open!
GitHub will show you which DNS records you need to create. Keep this page open - we'll use it in the next step when we configure your domain.

Add DNS Records at Your Domain Registrar

Now we need to tell your domain where to point. Log in to where you bought your domain (GoDaddy, Namecheap, etc.) and find the DNS settings. I'll show you exactly what to add.

üî∑ FOR SUBDOMAIN (like members.yourname.com):

Add a CNAME record with these settings:

Type:
CNAME
Host/Name:
members (or whatever subdomain you chose)
Points to/Value:
yourusername.github.io
TTL:
1 hour (or leave default)

üî∑ FOR ROOT DOMAIN (like yourname.com):

This requires A records AND optionally a www CNAME. Here's exactly what to do:

üéØ CRITICAL: GitHub Pages Official A Records

For GitHub Pages, these are the official A records you MUST use.
Set all four for best performance and reliability:

185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153

Add them in GoDaddy (or your registrar) like this:

Type:
A
Name:
@ (this means root domain)
Value:
185.199.108.153 (first IP - repeat for all four)
TTL:
1 hour
Repeat for All Four IP Addresses
You need to create 4 separate A records, one for each IP address. Just change the "Value" field each time. When you're done, you'll have 4 A records total.

Also, for your www version to work, add this CNAME:

Type:
CNAME
Name:
www
Value:
your-github-username.github.io
TTL:
1 hour
Save Your Changes & Be Patient
Make sure to save all your DNS changes! DNS can take anywhere from a few minutes to a few hours to update worldwide. Usually it works within an hour, so grab a coffee and give it some time. You can check your DNS propagation status at dnschecker.org

  4. Once that appears, check the box that says "Enforce HTTPS" (this gives you the secure üîí padlock)
  5. If HTTPS isn't available yet, wait another 15 minutes and try again

üîß Troubleshooting - We've Got You Covered

My GitHub URL works but my custom domain doesn't - what's wrong?

If your GitHub URL (https://yourusername.github.io/yourname-site) works but your custom domain doesn't, the issue is with your DNS records. Here's what to check:

  • For subdomain: Did you create a CNAME record (not an A record)?
  • For root domain: Did you add all four A records with the correct IPs?
  • Did you type yourusername.github.io correctly with no typos?
  • Have you waited at least 60 minutes after making DNS changes?

DNS propagation can take time. Check your status at dnschecker.org

My site shows a 404 error - help!

A 404 error usually means GitHub can't find your homepage file. Check these things:

  • Your main file MUST be named index.html (all lowercase)
  • The index.html file must be in the ROOT of your repository, not in a subfolder
  • Go to your repo and verify you can see index.html in the main file list
  • Make sure you enabled GitHub Pages from the correct branch (usually "main")

If the file is in a subfolder, move it to the root and recommit.

HTTPS isn't available - what should I do?

The "Enforce HTTPS" option only appears after your DNS is fully propagated and GitHub has verified your domain. Here's what to do:

  • Wait at least 60 minutes after adding your custom domain
  • Make sure your DNS records are correct and propagated
  • Try removing and re-adding your custom domain in GitHub Pages settings
  • Refresh the Settings ‚Üí Pages page - the option should appear within 24 hours

Be patient - GitHub needs to provision the SSL certificate, which happens automatically but takes time.

How long does DNS propagation take?

DNS propagation time varies, but here's what to expect:

  • Fastest: 5-15 minutes (if you're lucky!)
  • Typical: 30-60 minutes for most changes
  • Maximum: Up to 24-48 hours in rare cases

In most cases, your site will work within an hour. You can track propagation status at dnschecker.org by entering your domain name.

Can I use both www and non-www versions of my domain?

Absolutely! If you're using a root domain setup, both versions will work:

  • The 4 A records handle your root domain (yourname.com)
  • The www CNAME record handles the www version (www.yourname.com)
  • GitHub Pages automatically redirects between them

Just make sure you added both the A records AND the www CNAME as shown in Step 6.

My site looks broken or images aren't loading - why?

If your site structure looks wrong or images/styles aren't loading:

  • Check that all folders (/images, /css, /js) were uploaded to the correct location
  • File and folder names are case-sensitive - make sure they match exactly
  • Your HTML file paths should be relative (like images/logo.png)
  • Avoid absolute paths that point to your computer (like C:/Users/...)

Open your browser's Developer Console (F12) to see specific errors about missing files.
